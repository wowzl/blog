Python语言基于GUI开发的MVP框架

发表于 2018-06-14 | 分类于 python |

起个名HuLuWa(葫芦娃)

大学我写的第一个项目，和爱易相关的就是起的这个名，哈哈 现在也想不到其他名

开发背景

最近在用python写一个windows小软件，使用的是thinker库，虽然页面布局，样式有点low，但是基本功能是实现了。在开发的过程中共经历了大概四个阶段。

阶段一：初期开发，用了一个页面index.py

平时都用php开发，所以写的时候有浓厚的php风格，比如首页就起了个index.py.将所有逻辑代码写到了一个文件里，大概有3-5个类，直接实现了想要的功能。

阶段二：功能增多了，将index.py拆分了

大概查了5个左右文件，相当于将之前的类单独建立建了文件，都在同一个目录下面，页面文件也只有2个，一个首页，一个操作页面。嗯，也能愉快的使用

阶段三：简单用了下mvc框架，

将分别建了Models,Control,View,Lib,Helper五个目录，M层使用了orm。代码写了好几个文件目录。还是用web的感觉去开发的，很多地方只是为了实现功能，没有考虑优化等等的。写完感觉还是别扭，不是自己想要的。

最后经过简单思考，决定采用MVP重构

对mvp了解不是很深，看了几篇博客，觉得还挺试用现在的状况。mvc开发桌面软件中，初期感觉良好，后来发现，用web端的感觉去开发有点别扭，最简单的就是所有的windows端的桌面程序就像安卓和ios开发一样，是事件触发操作，请求数据。
所以边学习边搞搞 重构下啦

开发方式

简单分为架构逻辑和业务逻辑
架构：我与强哥商量来搞
业务：我来弄（强哥有空的也可以帮我搞搞哈）

开发目录

HuLuwa
├─api //URL、接口管理、网络请求封装类
├─mvp //项目主体
│ ├─base //基础类封装
│ ├─contract/ /契约类 用于统一管理view和presenter的接口
│ ├─model //M层-数据处理
│ ├─presenter //P层-逻辑业务处理
│ └─view// V层-页面渲染
└─utils//工具类
